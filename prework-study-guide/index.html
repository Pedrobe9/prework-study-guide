HTML

  • The head element contains information about the webpage.
  • Headings are size h1 (biggest) up to h6 (smallest).
  • Title element in head appears in tab.
  • style and base element defines the base URL for a webpage.
  • link, meta and script element embeds client-side scripts in the HTML document.
  • The body element represents the visible content shown to the user.
  • Header element appears at the top of the page and usually contains a navigation menu and/or introductory content. In our case, it holds an h1 heading, up to h6.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS: h1 style="color:red;"-Prework Study Guide-/h1 -- change the color of a single h1 heading
  • Internal CSS style sheet: -head> style> h1 { color: red;} /style> /head> -- all h1> elements on the webpage will be red/h1>
  • External CSS style sheet: link the file using a -link> element placed in the HTML's -head>: -head> -link rel="stylesheet" href="./assets/style.css" /> -/head>
  • Do Not Repeat Yourself (DRY) is a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds modifications in the current working branch; -A: add all changes.
  • git commit -m "add starter code": commit message
  • git pull origin main: receive a branch's modifications into the local environment
